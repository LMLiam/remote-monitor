# Code of Conduct

`remote-monitor` should be a welcoming project for contributors, users, testers, and maintainers.

## Expected Behavior

- Be respectful and constructive.
- Assume good intent while still being clear about bugs and risks.
- Keep discussions focused on the project.
- Give actionable feedback in issues and pull requests.
- Respect maintainer decisions about scope, security, licensing, and release direction.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Personal attacks or repeated hostile comments.
- Publishing private information without permission.
- Spam, trolling, or intentionally derailing project discussions.
- Pressuring maintainers to accept unsafe changes or unsupported platform scope.

## Enforcement

Maintainers may edit, hide, lock, or delete comments; close issues or pull requests; block users; or report behavior to GitHub when needed.

If you need to report a conduct issue, contact the project owner privately through GitHub.
